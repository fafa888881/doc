
一、对接说明

2.数据提交和返回类型：application/json
3.回调IP：本平台不提供IP加白，请以签名数据包为准。如果需要双重签名认证，请与客服联系
4.下单IP：贵司需要提供下单IP加白

二、签名算法
1.使用md5签名，结果为全小写
2.先将参数按照第一个字符的键值ASCII码递增排序（字母升序排序）
3.将排序后的参数与其对应值，组合成“参数=参数值”的格式，并且把这些参数用 & 字符连接起来，然后在末尾加上签名密钥。
4.将上一步生成的字符串进行md5加密并转小写，得到签名结果。
3.签名示例：

{
    "a": 1
    "b": "10008",
    "c": 1636972859708,
    "d": "dd"
}

拼接后得到：

a=1&b=10008&c=1636972859708&d=dd

假如签名密钥为：asdflkjgosiudfwqer，则将该密钥添加到字符串末尾，得到等签名字符串:

a=1&b=10008&c=1636972859708&d=ddasdflkjgosiudfwqer

然后进行md5加密并转换为小写
